# Contributing to OpenL2M

## Git Access

In order to contribute to OpenL2M you do not need to be a contributor on this Git project!

Please fork this repo on github.com, create a new branch, and submit back to the OpenL2M project from this fork.

## Code Style

Make sure you adhere to our coding style, and use black, ruff and pylint accordingly.
See details at https://github.com/openl2m/openl2m/blob/main/docs/code/code_style.rst

## PR's

Before you submit a PR, please open a GitHub issue/ticket to introduce and explain your contribution. Ie. is it a bug-fix, a new feature or something else! We can then discuss any details of the code before possible merging the PR from your fork into the repo development branch.

## Bugs

If you think you have found a bug, validate that you are *using supported devices*, and the *latest released version*. If so, and you still think you have found a bug, please create an issue/ticket with as much details as possible to replicate the problem.

If possible, you can also run OpenL2M in DEBUG mode, and capture all output. Submit in the ticket.
